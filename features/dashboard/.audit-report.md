# Nasaq Dashboard Visual Consistency Audit Report

## Scope of Audit
This audit covers the 8 feature pages in the `features/dashboard/*` directories:
- `features/dashboard/bookings/bookings-page.tsx`
- `features/dashboard/crm/crm-page.tsx`
- `features/dashboard/inbox/inbox-page.tsx`
- `features/dashboard/invoices/invoices-page.tsx`
- `features/dashboard/overview/overview-page.tsx`
- `features/dashboard/reminders/reminders-page.tsx`
- `features/dashboard/reports/reports-page.tsx`
- `features/dashboard/settings/settings-page.tsx`

## Inconsistencies Found

### 1. Card Padding Variations
Cards across pages use different internal paddings, leading to inconsistent whitespace.
- **bookings**: p-4
- **crm**: p-4, p-3
- **inbox**: No padded Card or external component
- **invoices**: No padded Card or external component
- **overview**: p-2, p-6
- **reminders**: p-5, p-4
- **reports**: p-6, p-5
- **settings**: No padded Card or external component

### 2. Heading Level Inconsistencies
Section headers inconsistently mix `h2` and `h3` tags.
- **bookings**: h2: 0, h3: 1
- **crm**: h2: 0, h3: 0
- **inbox**: h2: 0, h3: 0
- **invoices**: h2: 0, h3: 0
- **overview**: h2: 0, h3: 1
- **reminders**: h2: 1, h3: 1
- **reports**: h2: 4, h3: 0
- **settings**: h2: 0, h3: 0

### 3. Filter/Search Bar Placement
Search and filter components are not uniformly present or styled.
- **bookings**: Has Search/Filter
- **crm**: Has Search/Filter
- **inbox**: No Search/Filter
- **invoices**: Has Search/Filter
- **overview**: No Search/Filter
- **reminders**: No Search/Filter
- **reports**: No Search/Filter
- **settings**: No Search/Filter

### 4. Empty State Usage
Standard `EmptyState` component is not used consistently across all pages.
- **bookings**: Yes (Component)
- **crm**: Yes (Component)
- **inbox**: No / Inline
- **invoices**: Yes (Component)
- **overview**: No / Inline
- **reminders**: Yes (Component)
- **reports**: No / Inline
- **settings**: No / Inline

### 5. Section Spacing (space-y-* / gap-*)
Spacing utility classes vary wildly between feature pages.
- **bookings**: space-y-8, space-y-4, space-y-3, gap-4, gap-3, gap-2, gap-1, gap-1.5
- **crm**: space-y-1, space-y-2, gap-1.5, gap-4, gap-3, gap-2, gap-1, gap-6, gap-2.5
- **inbox**: gap-4, gap-2
- **invoices**: gap-1.5, gap-4, gap-3, gap-2, gap-1, gap-2.5
- **overview**: space-y-6, gap-4, gap-3, gap-2, gap-6
- **reminders**: space-y-6, space-y-3, space-y-1, gap-2, gap-6, gap-4, gap-3, gap-1, gap-1.5
- **reports**: gap-2, gap-4, gap-6
- **settings**: space-y-4, space-y-2, gap-1, gap-1.5, gap-4, gap-3

## Top Inconsistencies summary
1. **Card Padding:** The paddings range from `p-3`, `p-4`, `p-5`, `p-6` creating visual mismatch in content boundaries.
2. **Heading Levels:** Semantic HTML is broken as pages arbitrarily mix `h2` and `h3` for similar section titles.
3. **PageShell Usage:** `inbox` uses `height="viewport"` and `overview` uses `surface="gradient"`, while others are mostly standard.
4. **Section Spacing:** Values range heavily (`gap-2`, `gap-3`, `gap-4`, `gap-6`, `space-y-3`, `space-y-4`, `space-y-8`), resulting in unequal vertical rhythms.
