## 2025-02-12 - Recharts Inline Array Iteration Anti-Pattern
**Learning:** Initializing data arrays inline inside Recharts component props (like `data={[ { value: array.filter().length } ]}`) not only creates severe performance bottlenecks with repeated O(N) filtering, but it also creates a new object reference on every render. Because Recharts heavily relies on reference equality, this anti-pattern triggers deep internal re-renders of the entire chart on completely unrelated state updates (e.g., just typing into a search input on the same page).
**Action:** Always extract derived Recharts data arrays using `useMemo`. If multiple charts or stats depend on the same underlying dataset, compute them together in a single iteration pass rather than chaining multiple `.filter()` calls.
